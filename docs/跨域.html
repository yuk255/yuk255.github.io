项目的api请求时遇到的跨域问题解决 ​

什么是跨域？

当一个请求url的协议、域名、端口三者之间任意一个与当前页面url不同即为跨域。 跨域并不会阻止请求的发出，也不会阻止请求的接受，跨域是浏览器为了保护当前页面，你的请求得到了响应，浏览器不会把响应的数据交给页面上的回调，取而代之的是去提示你这是一个跨域数据。 为什么会出现跨域问题？ 出于浏览器的同源策略限制。同源策略（Sameoriginpolicy）是一种约定，它是浏览器最核心也最基本的安全功能，如果缺少了同源策略，则浏览器的正常功能可能都会受到影响。可以说Web是构建在同源策略基础之上的，浏览器只是针对同源策略的一种实现。同源策略会阻止一个域的。javascript脚本和另外一个域的内容进行交互。所谓同源（即指在同一个域）就是两个页面具有相同的协议（protocol），主机（host）和端口号（port）。

解决思路

  1. 前端配置代理
  2. 使用JSONP方案
  3. 通过Access-Control-Allow-Origin响应头，就告诉了浏览器。如果请求我的资源的页面是我这个响应头里记录了的"源"，则不要拦截此响应，允许数据通行
  4. 使用CORS插件，跨域资源共享

前期开发阶段:

  • 前端：在vue.config.js中配置代理，如下
js
devServer: {
    port: 8080,//它是用来修改你打开后的端口号的
    open: false,//值为 true的话，项目启动时自动打开到浏览器里边， false不会打开
    proxy:{
      '/api':{
          target:'https://localhost:3002',//跨域请求的公共地址
          changeOrigin:true, //是否开启跨域，值为 true 就是开启， false 不开启
          pathRewrite:{
              '^/api':''//注册全局路径， 但是在你请求的时候前面需要加上 /api  
          }
      },
    }
  }

在项目配置项中配置代理，注册网络请求路径为/api，在使用时只需要/api即可，不用在写详细的地址了 但是项目打包后开发环境的配置文件是不会打包进去，所以这时候就需要后端配合解决。

  • 后端：

在后端api的路由配置中，使用Route::group设置路由分组，将需要解决跨域的包裹在一个分组中，并调用allowCrossDomain类并配置 'Access-Control-Allow-Origin' => '*', *代表允许所有的请求，一般上线环境只需配置项目的域名即可。

php
<?php

use think\facade\Route;
Route::group(function () {

    Route::rule('login', '/account/login', 'get|post');
    
    Route::rule('register', '/register/index', 'get|post');
    
    Route::rule('user', '/user/index', 'get|post')
    ->middleware(\app\middleware\CheckToken::class);
    
    Route::rule('logout', '/logout/index', 'get|post');
    
    Route::rule('add', '/add/index', 'get|post')
        ->middleware(\app\middleware\CheckToken::class);
    
    Route::rule('sub', '/sub/index', 'get|post')
        ->middleware(\app\middleware\CheckToken::class);
    
    Route::miss('/miss/index');

})->allowCrossDomain([
    'Access-Control-Allow-Origin'        => '*',
]);
